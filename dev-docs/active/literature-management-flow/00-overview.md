## Goal
- 将文献管理从“导航占位”推进为可执行模块，建立「检索/导入 -> 去重入库 -> 标签与引用状态 -> 论文项目关联」闭环。

## Non-goals
- 不实现跨用户文献共享。
- 不在 M0 引入全文向量化与全文 RAG。
- 不在本任务中重构 M2~M8 模块。

## Context
- 需求与术语已明确要求项目级文献注册表、去重、来源追溯与摘要级 RAG。
- 当前后端仅在创建论文项目时校验 `initial_context.literature_evidence_ids` 非空，缺少完整文献注册表实体与 API。
- 桌面端已有「文献管理」导航入口，但尚未形成业务流程界面与交互。

## 文献管理流程梳理（目标态）
1. 检索/导入入口：用户输入检索条件或导入本地文献。
2. 规范化处理：统一抽取 DOI/arXiv/title/author/year/source_url 等字段。
3. 去重判定：按 DOI > arXiv > 标题+作者+年份执行多级去重。
4. 注册入库：写入文献注册表并记录来源、抓取时间、授权状态、标签、引用状态。
5. 关联项目：将候选文献挂载到 topic/paper 项目，生成 evidence 关联。
6. 反馈循环：支持刷新元数据、调整标签、更新引用状态，保留审计链路。

## Acceptance criteria (high level)
- [ ] 文献流程节点的输入/输出与失败路径完整定义。
- [ ] 可稳定导入并去重，且不会产生重复实体泛滥。
- [ ] 可把文献关联到论文项目并回读关联结果。
- [ ] M0 约束（摘要级 RAG、来源可追溯、授权边界）被明确并可验证。
